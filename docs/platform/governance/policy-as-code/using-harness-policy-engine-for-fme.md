Harness Policy As Code supports governance for Feature Management & Experimentation (FME) feature flags and feature flag definitions. You can define policies in Rego that are automatically evaluated on the **On Save** trigger whenever a feature flag or feature flag definition is created, updated, deleted, or archived.

For complete documentation including setup steps, input payload reference, and policy examples, see [Using Harness Policy As Code with FME](/docs/feature-management-experimentation/policies).

For FME-specific policy samples, see [FME Feature Flag policies](/docs/platform/governance/policy-as-code/sample-policy-use-case#fme-feature-flag-policies) and [FME Feature Flag Definition policies](/docs/platform/governance/policy-as-code/sample-policy-use-case#fme-feature-flag-definition-policies).
