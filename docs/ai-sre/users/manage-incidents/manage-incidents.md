# Active Incident Management

Once an incident is created — whether by you, a teammate, or automatically from a monitoring alert — the **Incident Details** page is your central workspace for driving it to resolution. 

This section covers everything from acknowledging a page to closing out the incident.

## Incident Lifecycle

| Phase | Task | Description |
| --- | --- | --- |
| Respond | [Acknowledge and triage an incident](./acknowledge-and-triage) | Open the incident, review the summary, acknowledge, and assess impact |
| Investigate | [Update incident details](./update-incident-details) | Edit fields, change status, and add key events as the situation evolves |
| Investigate | [Use the incident timeline](./use-the-incident-timeline) | Post updates, coordinate with responders, and review automated activity |
| Remediate | [Execute runbooks](./execute-runbooks) | Run automated and manual response procedures attached to the incident |
| Follow up | [Create and track action items](./create-and-track-action-items) | Capture follow-up tasks, assign owners, and track completion |
| Close | [Resolve and review an incident](./resolve-and-review) | Close the incident, review AI Scribe outputs, and contribute to post-incident learning |
