The Harness Web Application and API Protection Developer exam tests your knowledge and skills of the Harness Web Application and API Protection module.  

## Prerequisites

- Basic terminal skills
- Basic understanding of Harness and API Protection.

## Exam Details

Exam Duration: 90 minutes <br/>
Question Type: Multiple choice

| Covered Domain                                          | Percentage |
| ------------------------------------------------------- | ---------: |
| Understanding Platform Architecture & Deployment        |    **13%** |
| API Discovery, Inventory & Conformance Analysis         |    **17%** |
| Risk Assessment & Sensitive Data Discovery              |    **13%** |
| API Security Testing (XAST)                             |    **17%** |
| Threat Detection, Protection Policies & Runtime Defense |    **20%** |
| Threat Investigation & Incident Response                |     **7%** |
| Integrations, Automation & CI/CD                        |     **7%** |
| Operations, Reporting & Issue Management                |     **6%** |
| **Total**                                               |   **100%** |

## Exam Objectives

<details>
<summary>List of Objectives</summary>

The following is a detailed list of exam objectives:

| #       | Objectives                                                                                                                        |
| ------- | --------------------------------------------------------------------------------------------------------------------------------- |
| **1**   | **Understanding Platform Architecture & Deployment**                                                                              |
| **1.1** | Explain the core components of the Harness API Security platform, including Platform Agent, Tracing Agents, and Edge deployments. |
| **1.2** | Compare deployment options including Edge, Sidecar Proxy, Gateway Integration, eBPF Mirroring, and SPAN/Port Mirroring.           |
| **1.3** | Describe how API traffic is collected, processed, and analyzed throughout the platform.                                           |
| **2**   | **Discovering and Inventorying APIs**                                                                                             |
| **2.1** | Understand how APIs are automatically discovered and inventoried from runtime traffic.                                            |
| **2.2** | Identify APIs using HTTP methods, paths, and endpoint metadata.                                                                   |
| **2.3** | Interpret traces, spans, API inventory, and Application Flow Maps to understand application communication.                        |
| **3**   | **Analyzing API Posture & Conformance**                                                                                           |
| **3.1** | Compare runtime API behavior against OpenAPI specifications using Conformance Analysis.                                           |
| **3.2** | Differentiate Shadow APIs, Orphan APIs, and Endpoint Drift.                                                                       |
| **3.3** | Monitor API lifecycle changes using Posture Events and posture reporting.                                                         |
| **4**   | **Assessing API Risk & Sensitive Data**                                                                                           |
| **4.1** | Understand API Risk Scores and the factors that contribute to endpoint risk.                                                      |
| **4.2** | Detect sensitive information including PII and PCI data using built-in classifiers.                                               |
| **4.3** | Prioritize vulnerabilities using exposure, impact, sensitive data, and blast radius.                                              |
| **5**   | **Performing API Security Testing (XAST)**                                                                                        |
| **5.1** | Configure XAST scans, authentication methods, and scan settings.                                                                  |
| **5.2** | Differentiate XAST Live, XAST Replay, and security testing workflows.                                                             |
| **5.3** | Integrate Security Gates into CI/CD pipelines to prevent vulnerable deployments.                                                  |
| **6**   | **Detecting and Responding to API Threats**                                                                                       |
| **6.1** | Identify common API vulnerabilities including the OWASP API Top 10 (such as BOLA).                                                |
| **6.2** | Understand runtime threat detection using behavioral analytics, signatures, and threat intelligence.                              |
| **6.3** | Investigate Threat Actors and perform threat containment and response actions.                                                    |
| **7**   | **Configuring Protection Policies**                                                                                               |
| **7.1** | Differentiate WAF, API Protection, AI & LLM Protection, and Bot Protection policies.                                              |
| **7.2** | Configure policy evaluation order, exclusions, monitoring, and enforcement modes.                                                 |
| **7.3** | Apply runtime protection strategies and deployment best practices to minimize false positives.                                    |
| **8**   | **Automating Security Operations & Integrations**                                                                                 |
| **8.1** | Integrate API Security with GitHub Actions, Jenkins, GitLab, Azure DevOps, Jira, and other DevSecOps tools.                       |
| **8.2** | Manage issue lifecycle, risk acceptance, remediation workflows, and developer collaboration.                                      |
| **8.3** | Use dashboards, Security Performance Index, reporting, and Application Flow Maps for operational visibility.                      |



</details>
